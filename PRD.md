## 1. 제품 개요

### 1.1 한 줄 정의
유튜브 크리에이터와 광고주를 직접 연결하는 협찬 마켓플레이스.

### 1.2 해결하는 문제

| 대상 | 현재 문제 | ViewPoint 해결책 |
|---|---|---|
| **광고주** | 인플루언서 에이전시 수수료 20-30%, 긴 중개 기간 | 직접 제안 발송, 크레딧 기반 투명 과금 |
| **크리에이터** | 협찬 문의 채널 부재, 조건 협상 난이도 | 표준화된 광고 단가 공개, 수락/거절 단순화 |
| **시장** | 오픈 예정 채널 협찬 선점 불가 | 얼리버드 스폰서십 — 경쟁 서비스 없는 차별점 |

### 1.3 핵심 차별점
- **얼리버드 스폰서십**: 오픈 예정 채널(UPCOMING)에 사전 후원 → 채널 런칭 전 광고 선점
- **직접 거래**: 에이전시 없이 크리에이터-광고주 1:1 협상
- **투명한 단가**: 채널마다 광고 유형별 단가 공개

### 1.4 운영사 정보
- 운영: (주)라운드미디어 (Round Media)
- 플랫폼 브랜드: nextidealab.ai by (주)루시퍼
- 결제 수취: Dodo Payments (MoR, 글로벌 수취 가능)

## 2. 사용자 (User Personas)

### 2.1 크리에이터 (CREATOR)
- **정의**: 유튜브 채널을 운영 중이거나 오픈 예정인 콘텐츠 크리에이터
- **목표**: 협찬 광고주 발굴, 투명한 조건 협상, 안전한 수익 정산
- **pain point**: 협찬 문의를 DM/이메일로 받아 관리 어려움, 단가 기준 없음
- **행동 흐름**: 채널 등록 → 심사 통과 → 제안서 수신 → 수락/거절 → 정산

### 2.2 광고주 / 스폰서 (SPONSOR)
- **정의**: 유튜브 채널을 통해 브랜드를 홍보하려는 기업 또는 개인 마케터
- **목표**: 타겟 맞는 채널 탐색, 직접 제안서 발송, 협찬 집행 결과 관리
- **pain point**: 에이전시 수수료 부담, 채널 단가 불투명, 계약 절차 복잡
- **행동 흐름**: 회원가입(무료 3크레딧) → 채널 탐색 → 제안서 발송 → 크레딧 추가 구매

### 2.3 관리자 (ADMIN)
- **정의**: (주)라운드미디어 운영팀
- **목표**: 채널 품질 관리, 거래 모니터링, 정산 처리
- **행동 흐름**: 채널 심사(승인/반려) → 이상 거래 모니터링 → 정산 확인

## 4. 기능 요구사항

### 4.1 공개 페이지 (비로그인)

#### F-01: 메인 랜딩 페이지
- ShuffleHero: 4×4 채널 이미지 셔플 애니메이션 + 헤드카피
- ChannelGrid: 승인된 채널 목록 (카테고리 필터, ACTIVE/UPCOMING 필터, 검색)
- ChannelCard: 썸네일, 카테고리, 채널명, 구독자/조회수, 광고 단가, UPCOMING 진척도 바
- ChannelModal: 클릭 시 우측 슬라이드 패널 (상세 정보 + 제안서 보내기 CTA)

#### F-02: 안내 페이지
- 크리에이터 가이드 (`/guide/creator`): 등록 프로세스, 프리미엄 구독 안내
- 광고주 가이드 (`/guide/sponsor`): 크레딧 시스템, 얼리버드 스폰서십 안내
- 공지사항, FAQ, 이용약관, 개인정보처리방침

### 4.2 인증 (로그인/회원가입)

#### F-03: 회원가입
- 역할 선택 UI: CREATOR / SPONSOR 선택 (가입 전 반드시 선택)
- Google OAuth + 이메일/비밀번호 Credentials
- SPONSOR 가입 시: CreditBalance 자동 생성 (balance: 3)
- URL 쿼리 파라미터 `?role=creator|sponsor` 지원

#### F-04: 로그인
- 이메일 + 비밀번호
- 구글 소셜 로그인
- 로그인 후 역할별 자동 리다이렉트 (`/dashboard/creator` or `/dashboard/sponsor`)

### 4.3 크리에이터 기능

#### F-05: 채널 등록
- 채널 유형: ACTIVE (운영 중) / UPCOMING (오픈 예정)
- 필드: 채널명, 설명, YouTube URL, 카테고리, 태그, 업로드 주기
- ACTIVE 전용: 구독자수, 평균 조회수, 광고 단가 (JSON: integrated, shorts)
- UPCOMING 전용: 오픈 예정일, 펀딩 목표, 얼리버드 마감일, 얼리버드 단가
- 포트폴리오 이미지 3장 업로드 (Supabase Storage)
- 등록 즉시 `PENDING` 상태 → 관리자 심사 대기

#### F-06: 채널 관리
- 등록 채널 목록 조회 (상태 뱃지 포함)
- 채널 정보 수정 / 삭제

#### F-07: 제안서 관리 (수신)
- 받은 제안서 목록 (광고주, 채널, 금액, 상태, 날짜)
- 수락 / 거절 액션
- 수락 시 광고주에게 결제 링크 트리거

#### F-08: 수익 / 정산
- 총 수익 요약
- 정산 내역 테이블 (날짜, 광고주, 금액, 수수료 10%, 수령액, 상태)

### 4.4 광고주 기능

#### F-09: 채널 탐색 및 제안서 발송
- 랜딩 채널 그리드에서 탐색
- 채널 모달에서 "제안서 보내기" → 제안서 폼
  - 제목, 메시지, 광고 유형, 제안 금액, 희망 집행일
- 발송 시 크레딧 1 차감 (잔액 0이면 결제 페이지 리다이렉트)

#### F-10: 제안서 관리 (발신)
- 보낸 제안서 목록 (상태 칩: SENT/VIEWED/ACCEPTED/COMPLETED)
- 행 클릭 → 채널 모달

#### F-11: 크레딧 관리 및 결제
- 잔여 크레딧 표시
- 크레딧 패키지 구매 (Dodo Payments)
  - STARTER: 5크레딧 / ₩49,000
  - GROWTH: 20크레딧 / ₩149,000 (BEST VALUE)
  - PRO: 50크레딧 / ₩299,000
- 결제 내역 조회

### 4.5 어드민 기능

#### F-12: 채널 심사
- PENDING 채널 목록 (탭: PENDING / APPROVED / REJECTED)
- 승인 → `APPROVED`, 반려 → `REJECTED` + 사유 입력

#### F-13: 회원 관리
- 사용자 목록 검색
- 역할, 가입일, 채널수, 제안서수, 상태 표시

#### F-14: 정산 관리
- 전체 결제 내역
- 플랫폼 수수료 수익 합산

#### F-15: 어드민 통계 대시보드
- 총 사용자, 오늘 심사 대기, 이번달 거래액, 플랫폼 수수료 수익

## 5. 비기능 요구사항

### 5.1 성능
- 채널 목록 페이지 초기 로드 < 2초 (Vercel Seoul region)
- 이미지: Supabase Storage CDN 활용
- API 응답: < 500ms (Prisma 쿼리 최적화, 인덱스 필수)

### 5.2 보안
- 모든 API 라우트: 세션 + 역할 검증 필수
- Dodo Webhook: 서명(signature) 검증 필수
- 비밀번호: bcryptjs 해싱 (saltRounds: 10)
- 환경변수: `.env.local` 전용, 절대 커밋 금지

### 5.3 접근성 & UX
- 반응형: 모바일 / 태블릿 / 데스크탑
- 브루탈리즘 디자인: 그림자 없음, 곡률 최소화, 흑백 대비

### 5.4 에러 처리
- 모든 API: `{ success: boolean, data?: any, error?: string }` 표준 응답
- 크레딧 부족: 결제 페이지 자동 리다이렉트
- 권한 없음: 역할별 적절한 페이지 리다이렉트
